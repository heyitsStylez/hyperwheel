# Rename repo from rysk-pnl-tracker to hyperwheel

The repo was renamed from `heyitsStylez/rysk-pnl-tracker` to
`heyitsStylez/hyperwheel` to match the product name shown in the app, the
Vercel project name, and the live `hyperwheel.vercel.app` URL. The original
name reflected only the Rysk integration; the product now also covers
Hypersurface and is branded "HyperWheel" in-app.

GitHub's automatic URL/clone redirects from the old name remain in place
indefinitely, so external links, existing forks, and any cached references
continue to work without action.

## The non-reuse rule

**Never create a new repo named `rysk-pnl-tracker` on this account.** GitHub's
redirect from the old name to `hyperwheel` breaks the moment any other repo
claims that slug — clones, web links, and the version-link footer (ADR 0001)
would silently route to the wrong place. The old name is reserved by
omission, not by a marker file, so this rule has to live somewhere a future
maintainer will read it. Hence this ADR.

## What was updated in-repo

- `git remote origin` (local clone)
- `src/html/body.html` footer source link
- `CLAUDE.md` issue tracker reference
- Built artefacts (`hyperwheel.html`, `public/index.html`) regenerated by
  `build.py`
- In-flight issue bodies (#16, #17, #20) — URL strings updated for
  cleanliness, though GitHub's redirect would have covered them

## What was not changed

- Local clone directory name (`2-hyperwheel`) — independent of remote
- Vercel project — already named `hyperwheel`; the GitHub App link tracks
  repo by internal ID and updated automatically
- GitHub Actions workflows — use `${{ github.repository }}` context, no
  hardcoded name
